Plugin System

lib-plugin-abi

Stable ABI definitions for the universal Rust plugin system.

Overview

This crate provides FFI-safe types that plugins compile against. It uses abi_stable to ensure compatibility across Rust versions.

For Plugin Authors



use lib_plugin_abi::*;



static VTABLE: PluginVTable = PluginVTable {


    info: plugin_info,


    init: plugin_init,


    update: ROption::RNone,


    cleanup: plugin_cleanup,


    handle_message: ROption::RNone,


};



extern "C" fn plugin_info() -> PluginInfo {


    PluginInfo::new("vendor.my-plugin", "My Plugin", "1.0.0", "extension")


        .with_author("Your Name")


        .with_description("What my plugin does")


}



extern "C" fn plugin_init(_ctx: *mut PluginContext) -> i32 {


    0 // Success


}



extern "C" fn plugin_cleanup(_ctx: *mut PluginContext) {}



#[no_mangle]


pub extern "C" fn plugin_entry() -> *const PluginVTable {


    &VTABLE


}

Types

  • PluginInfo - Plugin metadata (id, name, version, type)
  • PluginVTable - Plugin interface (init, update, cleanup, handle_message)
  • PluginContext - Context passed to plugin functions
  • HostVTable - Host callbacks available to plugins
  • PluginError - FFI-safe error type
