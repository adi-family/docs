Tarminal Signaling Server

WebSocket relay server for global sync between Tarminal devices.

Deploy to Fly.io



Install flyctl if not already installed


brew install flyctl



Login to Fly.io


fly auth login



Create and deploy the app


cd crates/tarminal-signaling-server


fly launch --name tarminal-signal --no-deploy


fly deploy

Your server will be available at wss://tarminal-signal.fly.dev/ws

Deploy to Other Platforms

Docker



docker build -t tarminal-signaling .


docker run -p 8080:8080 tarminal-signaling

Railway / Render

Just connect your repo and point to this directory. The Dockerfile will be auto-detected.

Update App URLs

After deploying, update the server URL in:

  • TarminalNativeiOS/Sources/WebSyncServiceiOS.swift
  • TarminalNative/Sources/WebSyncService.swift

Replace wss://tarminal-signal.fly.dev/ws with your actual server URL.

Protocol

The server uses JSON over WebSocket. Message types:

  • register - Register device with server
  • create_pairing_code - Generate 6-char pairing code
  • use_pairing_code - Pair with another device
  • sync_data - Forward sync payload to paired device

See src/main.rs for full protocol details.
