Why ADI?

Building AI agents that actually work requires more than just an LLM. You need infrastructure for understanding code, managing context, executing tasks, and connecting to the world.

01

Deep Code Understanding

Semantic code indexing with embeddings. Your agents don't just see code - they understand it. Search by meaning, not just keywords.

02

Persistent Knowledge

Graph-based knowledgebase that grows with your project. Agents remember context across sessions and share insights.

03

Isolated Execution

Docker-based task executor. Run agent workloads in isolated containers with full control over environment and resources.

04

Universal Interfaces

Every component exposes CLI, HTTP, and MCP interfaces. Integrate with any tool, any workflow, any LLM.

Installation

Get ADI running in seconds. Choose your preferred installation method.

Terminal 
# Install the ADI CLI
cargo install adi-cli

# Verify installation
adi --version
adi 0.8.4

# Or build from source
git clone --recursive https://github.com/adi-family/cli
cd cli && cargo build --release

Platforms
macOS, Linux, Windows (WSL2)

Quick Start

Get your first agent running in under 5 minutes.

1

Initialize your project

Create an ADI workspace in your project directory. This sets up the configuration and prepares your codebase for indexing.

adi init
2

Index your codebase

Build a semantic index of your code. ADI parses your files, generates embeddings, and creates a searchable knowledge graph.

adi index
3

Start the agent loop

Launch an autonomous agent with access to your indexed codebase. The agent can read, understand, and modify your code.

adi agent start
4

Connect via MCP

Expose ADI capabilities to any MCP-compatible client. Claude, Cursor, and other tools can now use your agent infrastructure.

adi mcp serve

Architecture

ADI follows a layered architecture. Each layer builds on the previous, creating a powerful and flexible system.

Interface Layer

CLI, HTTP API, MCP Servers - Multiple ways to interact with every component

Plugin Layer

Dynamic plugins with service registry, hot-reloading, and dependency management

Component Layer

Indexer, Knowledgebase, Task Manager, Agent Loop - The building blocks

Core Layer

Shared libraries for embeddings, migrations, color, animation, and more

Plugins

Extend ADI with plugins. The plugin system uses a stable ABI for binary compatibility across versions.

# List available plugins from registry
adi plugin list

NAME              VERSION   DESCRIPTION
code-indexer      0.3.0     Semantic code indexing with embeddings
task-manager      0.2.1     Task tracking and dependency management
git-tools         0.1.5     Git operations and history analysis
llm-providers     0.4.0     Multi-provider LLM support

# Install a plugin
adi plugin install code-indexer

# View installed plugins
adi plugin installed

Plugin Capabilities

  • Service Registry - Plugins can provide and consume services from other plugins
  • Hot Reloading - Update plugins without restarting the host process
  • Dependency Management - Automatic topological loading based on plugin dependencies
  • Signature Verification - Ed25519 signatures ensure plugin authenticity

Services

The service registry enables inter-plugin communication with version-aware resolution.

# List registered services
adi services

SERVICE                 VERSION   PROVIDER
indexer.search          1.0.0     code-indexer
indexer.embed           1.0.0     code-indexer
tasks.create            1.0.0     task-manager
tasks.list              1.0.0     task-manager
llm.complete            2.0.0     llm-providers
llm.embed               2.0.0     llm-providers

# Call a service directly
adi service call indexer.search --query "authentication handler"

Task Manager

Structured task tracking with dependencies, priorities, and agent integration.

  --priority high \
  --tags backend,security

# List tasks

# Assign to agent
adi agent assign task_01

CLI Commands

The ADI CLI provides access to all functionality. Every command supports --help for detailed usage.

Command Description
adi init Initialize ADI in current directory
adi index Index codebase for semantic search
adi search <query> Semantic search across indexed code
adi agent start Start the agent loop
adi plugin <cmd> Manage plugins (install, remove, list)
adi services List registered services
adi task <cmd> Task management (create, list, update)
adi executor <cmd> Docker job execution
adi mcp serve Start MCP server
adi update Self-update to latest version

MCP Servers

Every ADI component exposes an MCP (Model Context Protocol) server. Connect your favorite AI tools directly to ADI infrastructure.

adi-indexer-mcp

Semantic code search, symbol lookup, and dead code analysis via MCP.

adi-tasks-mcp

Task creation, updates, and queries. Let agents manage their own todos.

adi-knowledgebase-mcp

Store and retrieve knowledge. Agents can remember across sessions.

adi-agent-loop-mcp

Control agent execution. Start, stop, and monitor agent loops.

claude_desktop_config.json 
{
  "mcpServers": {
    "adi-indexer": {
      "command": "adi",
      "args": ["mcp", "indexer"]
    },
    "adi-tasks": {
      "command": "adi",
      "args": ["mcp", "tasks"]
    }
  }
}

Configuration

ADI is configured via adi.toml in your project root.

adi.toml 
[project]
name = "my-awesome-project"
version = "0.1.0"

[indexer]
include = ["src/**/*.rs", "lib/**/*.rs"]
exclude = ["target/**", "tests/**"]
embedding_model = "text-embedding-3-small"

[agent]
provider = "anthropic"
model = "claude-sonnet-4-20250514"
max_iterations = 50

[executor]
docker_host = "unix:///var/run/docker.sock"
default_timeout = "5m"
max_concurrent_jobs = 4

[plugins]
registry = "https://plugins.adi.dev"
auto_update = true
